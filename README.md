# Astrology & Horoscope — Flutter App

A complete Flutter source project for a daily horoscope / zodiac astrology app, matching the feature set:
daily/weekly/monthly horoscopes, zodiac sign profiles, birth chart summary, love compatibility checker,
lucky number/color, daily notifications, and an AI astrology assistant chat.

## What's included

```
lib/
  main.dart                       — app entry point
  theme/app_theme.dart            — dark "deep space" theme, colors, fonts
  models/                         — ZodiacSign, Horoscope, CompatibilityResult
  data/
    zodiac_data.dart              — all 12 signs (dates, traits, colors, element)
    horoscope_generator.dart      — local horoscope + compatibility logic
  services/
    user_provider.dart            — profile state, persisted via shared_preferences
    ai_service.dart                — AI assistant service (see "Wiring the AI Assistant" below)
    notification_service.dart      — daily local notification scheduling
  screens/                         — Splash, Onboarding, Home (bottom nav),
                                       Horoscope, Zodiac Browse/Detail,
                                       Compatibility, AI Assistant, Profile, Birth Chart
  widgets/                         — reusable UI components
pubspec.yaml                       — dependencies
analysis_options.yaml              — lint rules
```

This is a **lib-only** Flutter project — the `android/`, `ios/`, `web/` platform folders
are generated by the Flutter tool itself (see setup below) rather than hand-written, since
they're large, mostly-boilerplate, and tied to your specific package name / bundle ID.

## Setup (one-time)

1. **Install Flutter** if you haven't already: https://docs.flutter.dev/get-started/install
2. Unzip this project, then from inside the folder run:
   ```bash
   flutter create --org com.yourcompany --project-name astrology_app .
   ```
   This generates the `android/`, `ios/`, `web/`, `macos/`, `linux/`, `windows/` folders
   around the existing `lib/` and `pubspec.yaml` **without overwriting them**.
   Replace `com.yourcompany` with your actual reverse-domain identifier — this becomes
   your Android `applicationId` and iOS bundle identifier.
3. Install dependencies:
   ```bash
   flutter pub get
   ```
4. Run it:
   ```bash
   flutter run
   ```

## Wiring the AI Assistant (important)

`lib/services/ai_service.dart` currently runs on **local canned responses** so the app
is fully demoable offline. To make it a real AI assistant:

1. Stand up a small backend endpoint (Vercel/Cloud Function/Node server — anything) that:
   - Accepts `{ sign, question }`
   - Calls an LLM API (e.g. Anthropic's Messages API) server-side, where your API key
     is safely hidden
   - Returns `{ reply: "..." }`
2. In `ai_service.dart`, set `_backendUrl` to your endpoint and flip `useBackend = true`.

**Never embed an LLM API key directly in the Flutter app** — anything shipped in a mobile
binary can be extracted and abused.

## Wiring real daily notifications

`notification_service.dart` schedules a local notification (no server needed) at a fixed
time. Call `NotificationService.scheduleDailyReminder(hour: 9, minute: 0)` after the user
opts in (already wired into the Profile screen's toggle). For Android 13+, you'll also
need to request the `POST_NOTIFICATIONS` runtime permission — the `flutter_local_notifications`
package docs cover this step for the latest Android target SDK.

## Making horoscope content richer

`horoscope_generator.dart` currently generates readings from local template pools, seeded
by date + sign so they're stable per day/week/month rather than random on every open. For
production-quality content you have two solid options:

- **Astrology content API** — several third-party providers offer daily horoscope feeds
  you can fetch and cache.
- **LLM-generated copy** — reuse the same backend pattern as the AI Assistant to generate
  horoscope text server-side each morning and cache it, rather than calling an LLM live
  on every screen load.

## Real birth charts (Sun/Moon/Rising)

The Birth Chart screen currently shows a simplified, deterministic estimate for Moon and
Rising signs because true natal chart calculation requires **birth time and location**,
not just date, plus an ephemeris (planetary position) calculation — typically done with
a library like the Swiss Ephemeris or a dedicated astrology API. The code is structured
so you can swap in a real calculation inside `birth_chart_screen.dart` once you have that
data and a calculation source.

## Publishing checklist

- **App icon & splash**: replace the default Flutter icons — the `flutter_launcher_icons`
  and `flutter_native_splash` packages automate this from a single source image.
- **App name / bundle ID**: set via the `flutter create --org` step above, or edit
  `android/app/build.gradle` (`applicationId`) and `ios/Runner.xcodeproj` (Bundle Identifier)
  directly.
- **Privacy policy**: required by both app stores since this app collects birth date —
  make sure your store listing links to one.
- **Store listing copy**: see the separate `astrology-app-store-listing.md` file already
  generated for you, with App Store/Google Play field-by-field text and character counts.
- **Build release binaries**:
  ```bash
  flutter build appbundle   # Android, for Google Play
  flutter build ipa         # iOS, for App Store Connect (requires macOS + Xcode)
  ```

## Notes on the "AI Astrology Assistant" and "Accurate" claims

Keep in mind astrology predictions are inherently subjective/entertainment content —
many app stores require this to be disclosed in your listing (e.g. "for entertainment
purposes"), and some regions have specific rules about health/financial claims in
astrology apps. Worth checking your target stores' current policies before publishing.
